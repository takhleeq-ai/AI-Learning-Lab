# AI Learning Lab

A public, experiment-driven journey into machine learning and applied AI.

This repository is where I learn by doing: running small, focused experiments, documenting what I observe, and building practical intuition step by step.

Rather than trying to jump straight to advanced systems, the goal here is to build from first principles and make the learning visible.

---

## Why this repository exists

My professional background is in fintech and payments infrastructure, including work across the Apple Tap to Pay ecosystem and Lloyds Banking Group.

Most of my experience has been in regulated systems, where reliability, explainability, evaluation, and failure modes matter more than novelty.

This repository extends that systems thinking into machine learning.

I am using it to:

- learn core AI and ML concepts properly
- test ideas through small practical experiments
- document what worked, what did not, and what changed my understanding
- create a public record of steady technical progress

---

## Learning approach

I am not treating this as a tutorial collection.

Instead, I am working through a series of small experiments, each designed to answer a specific question.

Examples include:

- how a basic classifier works
- why accuracy can be misleading
- how precision and recall change model evaluation
- what happens when classes are imbalanced
- how model evaluation connects to real-world decision systems

Each experiment is intentionally scoped to be small enough to complete, understand, and explain clearly.

## Current focus

The current focus is on early supervised learning foundations, especially:

- classification basics
- model evaluation
- imbalanced datasets
- confusion matrices
- precision and recall

The aim is not just to get models running, but to understand how to judge them properly.

## How to read this repository

Each experiment lives in its own folder and usually contains:

- `experiment.ipynb` — the notebook for the practical experiment
- `notes.md` — a short written summary of the objective, method, and takeaway

Over time, this repository should become a readable record of progression rather than a dump of disconnected files.
